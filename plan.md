# 头号玩家 (Ready Player One) - 核心三位一体架构 (v3.1)

> [!IMPORTANT]
> **单一模型原则 (Single Model Principle)**: 
> 从 v3.1 开始，系统全面废弃“多模型并发猜测”的陈旧方案。我们采用单一的 **Super Brain YOLO** 模型
> 整合 Player、Monster、UI 等所有要素。这不仅将推理延迟降低了 40% 以上，更通过全局 NMS 彻底
> 解决了角色与怪物的识别冲突。

根据我们在实战中遇到的瓶颈，以及当前最优的 Agentic AI 设计理念，我们将彻底重构整个智能体的架构。不再使用笨重的 YOLO 训练，也不依赖不带脑子的像素级瞎走。

我们将整个智能体浓缩为最纯粹的三个能力维度：**看 (See)、按 (Act)、感 (Sense)**。

---

## 一、看 (See)：无感知的全天候“天眼”

我们需要在不影响人类玩家正常使用电脑（比如看网页、看电影）的情况下，拿到游戏的画面。
*   **核心痛点**：传统截图常常需要窗口置于前台，或者最小化后画面就会卡死/黑屏，导致脚本宕机。
*   **解决方案 (PrintWindow API / Desktop Duplication)**：
    *   通过 Windows 系统的底层图形接口（如 `PrintWindow` 加上 `PW_RENDERFULLCONTENT` 标志，或 DXGI 抓取）。
    *   **最终效果**：无论游戏是在后台被其他窗口完全遮挡，甚至是处于某些最小化状态，AI 依然能以稳定的 FPS（5-10帧即可）抽取到完整的 1600x900 画面，且完全不干扰用户的光标和当前焦点。

---

## 二、按 (Act)：幽灵般的“物理外挂”

解决“按”的核心是彻底隔离人类输入与 AI 输入。
*   **核心痛点**：普通的 `pyautogui` 会抢走人类的鼠标和键盘焦点，导致后台挂机成为笑话。
*   **解决方案 (DirectInput 后台穿透注入)**：
    *   已经验证过的 `AttachThreadInput` 挂载游戏线程 + 模拟 Scan Code 发送（或结合更底层的虚拟驱动 / PostMessage 高级封装）。
    *   **最终效果**：当你在一边用浏览器回复邮件时，AI 可以在后台稳定地向 MapleStory 进程注入 `[向左走] -> [跳跃] -> [攻击]` 的硬件级扫描码（Scan Codes）。游戏人物在疯狂打怪，而你的物理键盘和鼠标完全正常工作，互不干扰。

---

## 三、感 (Sense)：基于大模型 (Gemini 3.1) 的“降维建图”引擎

这是整个 V3.0 最具颠覆性的核心。抛弃以往在本地苦哈哈地标注数据、训练模型（如 YOLO）那种又慢又差的土办法。我们直接运用最先进的多模态大语言模型（如 Gemini VLM）作为智能体的“造物主大脑”。

*   **核心痛点**：每次换新地图、遇到新怪物，传统 CV 和小模型都会“瞎掉”，需要大量手动调参或重新训练。不知道哪是台阶、容易卡死在角落。
*   **解决方案 (基于 LLM 的 zero-shot 场景解析与 NavMesh 构建)**：
    1.  **初见新图 (The First Glance)**：当 AI 走到一个前所未见的全新地图时，立刻挂起动作，截取几张有代表性的画面。
    2.  **呼叫高维大脑 (Gemini Vision)**：将高清截图通过 API 发送给多模态大模型。同时下发 Prompt 设定（例如：“你是一个冒险岛顶级玩家，请告诉我图里哪个区域是怪物，哪些横条是人物可以踩踏的台阶/平台，哪里是梯子/绳子，请返回精确的 JSON 坐标”）。
    3.  **图鉴入库 (Monster Database)**：大模型返回类似 `{"怪物": ["红蜗牛", 坐标箱...]}`。AI 顺手将其裁剪并归档入本地 SQLite/特征库，后续在本地就能极速识别，无需每次都调 API。
    4.  **平台建模 (NavMesh / 傻瓜挂式地形图)**：最核心的一步！大模型返回 `{"台阶": [[x1,y1, x2,y2], ...]}`。AI 会把这些信息在内存里搭建成类似“傻瓜外挂（脱机挂）”那种内部坐标网格。
    5.  **降维打击执行 (Topological Pathfinding)**：当地形建好后，它不再是一张图片。AI 要走到右上角打怪，只需使用 A* 算法在它刚才建立的“台阶节点(Nodes)”和“跳跃路径(Edges)”之间一算，瞬间得出底层命令：`[右走5步] -> [上跳] -> [左走] -> [攻击]`。

---

## 接下来该怎么做？(Action Items)

我们既然确立了**“看、按、感”**三位一体，那么重构的第一刀应该斩向最难也是最有价值的：**【感 (Sense)】—— 大模型建图引擎**。

我们只需拿一张当前截下的复杂地形游戏画面（比如带有怪物、多层台阶的 1600x900 截图），写一个测试脚本发给大模型（如 Gemini），看看它返回解析出地形平台和怪物的精准度如何。一旦大模型能够成功吐出“地形可行走平台的线条坐标”和“怪物坐标”，我们这个 V3.0 宏伟蓝图的任督二脉就彻底打通了！
